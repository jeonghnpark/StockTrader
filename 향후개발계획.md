# 주식 포트폴리오 트래커 - 향후 개발 계획

본 문서는 1차 개발 완료 후, 향후 포트폴리오 트래커를 고도화하기 위한 2차 개발의 핵심 요구사항을 정리한 것입니다.

## 1. 속성(Attribute) 정의 및 관리 고도화
데이터 정합성을 높이고 다차원 분석을 위해 주식 및 계좌에 고정된 속성(상수)을 부여합니다.

* **1.1. 주식 통화 속성 구분:**
  * 각 종목이 USD 노출 자산인지, KRW 노출 자산인지 명확히 구별하는 속성 추가.
* **1.2. 주식 자산군 속성 구분:**
  * 각 종목을 `Index`(지수형), `Share`(개별주식), `Bond`(채권형) 등으로 분류하는 속성 추가.
* **1.3. 계좌 통화 속성 관리:**
  * 계좌별로 거래 가능한 통화를 명확히 구별하여 상수(Constant)로 취급하고, 매매 입력 시 데이터 정합성 검증에 활용.

## 2. 다차원 포트폴리오 통계 (Subsum)
* **2.1. 속성별 소계(Subsum) 산출:**
  * 위에서 정의한 속성(통화별, 자산군별, 계좌별)을 기준으로 포트폴리오의 전체 통계를 그룹화하여 일목요연하게 볼 수 있는 대시보드 화면 구성.
  * 예: "전체 자산 중 USD 노출 비중", "Index vs 개별주식 비중" 등.

## 3. 종목별 상세 분석 뷰
* **3.1. 종목 상세 페이지/모달:**
  * 특정 종목을 선택했을 때, 해당 종목에 대한 전체 투자 히스토리(매수/매도 타임라인)를 한눈에 볼 수 있는 기능 추가.
  * 종목별 누적 성과(실현 손익, 평가 손익 추이 등)를 시각화하여 제공.

## 4. 매매 사이클(Cycle) 기반 수익률 계산 고도화
* **4.1. 전량 매도 후 재매수 로직 (Reset):**
  * 매매 날짜를 감안하여, 특정 주식을 전량 매도(보유 수량 0)한 후 다시 재매입하는 경우를 파악.
  * 재매입 시점부터 매수 단가(Average Cost)를 새롭게 산정(Reset)하고, 평가 손익 또한 해당 사이클에 맞춰 새롭게 계산하도록 로직 수정.

## 5. 배당금 수익 자동화
* **5.1. 배당 정보 수집 및 반영:**
  * 매매 날짜와 보유 기간을 감안하여 해당 종목의 배당락일 및 배당금 지급 정보를 외부 API(yfinance 등)를 통해 수집.
  * 수집된 배당금을 매매 수익(배당 수익) 항목으로 별도 산정하여, 총 포트폴리오 수익에 합산해주는 기능 구현.
